Bonus Section - QnA

We know that, Javascript is a single threaded, high level,interpreted, just-in-time compiled programming language. It means it can only execute one command at a time and also in a specific order. That means it can go to next line if current line has been executed successfully. Everything of javascript happens in a execution context. Every browser like google chrome, firefox etc has javascript engine. For example google chrome has v8 engine. This engine takes the javascript syntax which is currently in human readable format and converts it into machine code(0,1) format as machine can not understand anything but 0,1.

Since javascript is single threaded, it is able to execute one command at a time. Suppose there are two button in a page. Now what if the user press these two button quickly one after another? Well, when one button is clicked, javascript starts doing that button's operation and as the second button is clicked also, javascript won't stop execution first button operation. Rather it should keep doing operation of that first button and in the mean time all the other operation are listed/noted on a queue. Javascript takes a note what to do after what and so on. It checks the queue and keep working until the queue is empty. Any event won't be executed before completing the previous event.
